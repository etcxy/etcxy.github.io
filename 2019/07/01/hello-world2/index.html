2019-07-01 种类 大数据 评论

1. 安装Cloudera Manager Server&Agent

  1. 拷贝cloudera-manager-*_x86_64.tar.gz到所有Server Agent节点
  2. 创建cm目录：
    mkdir /opt/cloudera-manager
  3. 解压cm压缩包：
    tar xvzf cloudera-manager*.tar.gz -C /opt/cloudera-manager

2. 创建用户cloudera-scm（所有节点）

  • cloudera-scm用户说明，摘自官网：
  • 执行：

3. 配置CM Agent

修改文件

/opt/cloudera-manager/cm-*/etc/cloudera-scm-agent/config.ini 中server_host以及server_port

server_host 改为主节点 server_port 端口号不变

4. 配置CM Server的数据库

  1. 将驱动包拷贝到目录下（注意拷贝过去的驱动包名字一定要和下边的一样，否则会报错）：

cp mysql-connector-java-5.1.31-bin.jar /usr/share/java/mysql-connector-java.jar

  1. mysq上创建临时用户

执行：

mysql> grant all on *.* to 'temp'@'%' identified by 'temp' with grant option;

cd /opt/cloudera-manager/cm-*/share/cmf/schema

./scm_prepare_database.sh mysql -h myhost1.sf.cloudera.com -utemp -ptemp --scm-host myhost2.sf.cloudera.com scm scm scm

（对应于：数据库类型、数据库服务器、用户名、密码、CMServer所在节点…….）

例如：

./scm_prepare_database.sh mysql -h node01 -utemp -ptemp --scm-host node01 scm scm scm

遇到错误:

解决方案：
检查hosts表，是否添加相关IP地址。

mysql> drop user 'temp'@'%';

若上步失败或过程中操作中断，删除所有库、重头来过/(ㄒoㄒ)/~~

5. 创建Parcel目录

  1. Manager节点创建目录 /opt/cloudera/parcel-repo

$> mkdir -p /opt/cloudera/parcel-repo

  1. 修改权限

$> chown cloudera-scm:cloudera-scm /opt/cloudera/parcel-repo

  1. 将下载好的文件
    （CDH-*.parcel CDH-*.parcel.sha manifest.json）拷贝到该目录下。

  2. Agent节点创建目录 /opt/cloudera/parcels

$> mkdir -p /opt/cloudera/parcels

  1. 修改权限

$> chown cloudera-scm:cloudera-scm /opt/cloudera/parcels

6、启动CM Server&Agent服务

  1. 执行：
  • Server：/opt/cloudera-manager/cm-*/etc/init.d/cloudera-scm-server start
  • Agents：/opt/cloudera-manager/cm-*/etc/init.d/cloudera-scm-agent start

2.访问：http://ManagerHost:7180，
若可以访问（用户名、密码：admin），则安装成功。

Manager启动成功需要等待一段时间，过程中会在数据库中创建对应的表需要耗费一些时间。

CDH安装失败后，重新安装

  1. 主节点关闭cloudera-scm-server和所有节点关闭cloudera-scm-agent

  2. 删除Agent节点目录

rm -rf /opt/cm-*

rm -rf /opt/cloudera

  1. 删除主节点 /var/log/ /var/lib/下cloudera相关的目录
  1. 清空CM数据库

进入Mysql数据库，然后drop database cm;

删除各节点namenode和datanode节点信息

rm -rf /opt/dfs/nn/* rm -rf /opt/dfs/dn/*

  1. 重新安装CDH
    主节点解解压：tar xzvf cloudera-manager*.tar.gz将解压后的cm-*和cloudera目录放到/opt目录下。
